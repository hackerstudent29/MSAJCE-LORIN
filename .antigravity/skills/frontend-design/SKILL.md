## SECTION A — WEBSITE TYPE DETECTION & BEHAVIOR
═══════════════════════════════════════════════════════════════

### BUSINESS / CORPORATE
- Clean hierarchy, trust signals, CTA-first
- Above-fold value prop in under 3 seconds
- Sections: Hero → Problem → Solution → Social Proof → CTA
- Mobile: single column, CTA button full-width and sticky

### CREATIVE / PORTFOLIO
- The design IS the product — go bold
- Grid-breaking layouts, overlapping elements, large type
- Mobile: simplify grid to single column, keep bold typography
- Cursor effects desktop-only — never on touch

### SCROLLYTELLING / STORYTELLING
- Narrative drives layout — content reveals as user scrolls
- Desktop: pin sections, parallax layers, progress-driven animation
- Mobile: convert pinned sections to stacked vertical flow
- Story readable without JavaScript always

### EDITORIAL / MAGAZINE
- Typography is the hero — type scale is the layout
- Desktop: asymmetric grids, full-bleed imagery
- Mobile: single column, large pull quotes, stacked spreads

### TECH / SAAS / STARTUP
- Feature grids, pricing tables, data visualization
- Desktop: multi-column feature grids, side-by-side comparisons
- Mobile: single-column stacked, pricing cards full-width

### HISTORY / HERITAGE THEME
- Aged textures, serif typography, timeline layouts
- Mobile: vertical timeline, full-width images, larger text

### FUTURE / SCI-FI THEME
- HUD interfaces, neon accents, terminal aesthetics
- Mobile: reduce density, keep glow effects, simplify grids

### E-COMMERCE / PRODUCT
- Product hero — full-bleed imagery
- Mobile: bottom sticky CTA bar, swipeable image gallery
- Touch: swipe gestures for product images mandatory

### DASHBOARD / DATA
- Desktop: sidebar + main content layout
- Tablet: collapsible sidebar or bottom tab navigation
- Mobile: bottom tab nav, cards stack vertically, charts scroll horizontally

### INTERACTIVE / GAME-LIKE
- 60fps animations — non-negotiable
- Touch equivalents for all mouse interactions
- Mobile: larger tap targets, simplified controls

### EDUCATIONAL / INSTITUTIONAL
- Accessibility-first, mobile-first — students use phones
- Mobile: large text, clear hierarchy, bottom navigation

### EVENT / LANDING PAGE
- One goal — conversion
- Mobile: full-width CTA, minimal nav, urgency above fold

## SECTION B — DESIGN SYSTEM LIBRARY
═══════════════════════════════════════════════════════════════

### APPLE STYLE (SF / Cupertino)
- Colors: Pure white, system gray scale, single blue accent #0071e3
- Typography: -apple-system, BlinkMacSystemFont stack
- Radius: 12–18px cards, 10px buttons
- Shadows: 0 4px 24px rgba(0,0,0,0.08)
- Motion: Spring physics — ease-in-out, never linear
- Glass: backdrop-filter blur(20px) on navbars
- Mobile: Respect iOS safe areas (env(safe-area-inset-*))
- Rule: Generous whitespace. If it feels empty, it's right.

### ANDROID MATERIAL 3 (Material You)
- Colors: Dynamic tonal palettes from seed color
- Typography: Google Sans / Nunito
- Components: FABs, chips, filled buttons, outlined cards
- Elevation: Tonal color shifts not shadows
- Mobile: Bottom navigation bar, FAB bottom-right
- Shapes: Rounded 28px+ — pill buttons, squircle cards
- Rule: Color-forward. Surface variants carry meaning.

### GLASSMORPHISM
- Background: Always a rich gradient or image behind
- Cards: background rgba(255,255,255,0.1), backdrop-filter blur(16–24px),
         border 1px solid rgba(255,255,255,0.2)
- Mobile: Reduce blur to 12px (performance), ensure text contrast
- Rule: Depth through blur, not shadow.

### NEUMORPHISM / SOFT UI
- Background: Single mid-tone (#e0e5ec)
- Shadows: Dual — light top-left, dark bottom-right
- Mobile: Increase shadow contrast slightly for outdoor readability
- Rule: Subtle. Overuse kills it.

### SKEUOMORPHISM
- Materials mimicking real objects: leather, paper, metal, wood
- Mobile: Simplify textures for performance, maintain material feel
- Rule: Commit fully or it looks like a mistake.

### BRUTAL MINIMALISM
- Max 2 colors. Extreme scale contrast. Uncomfortable empty space.
- Mobile: Scale maintains — single column, huge type still works
- Rule: Remove everything. Then remove more.

### BRUTALISM / RAW
- Ugly is intentional — broken grids, clashing colors
- Mobile: Controlled chaos — still readable, still intentional
- Rule: Intentionally broken, never accidentally broken.

### LUXURY / EDITORIAL
- Palette: Ivory, charcoal, gold #c9a84c, deep burgundy
- Typography: Cormorant, Playfair Display, Didot — thin weight
- Mobile: Generous padding maintained, full-bleed hero images
- Rule: Never rush. Luxury takes its time.

### ORGANIC / NATURAL
- Earth tones, soft blobs, hand-drawn strokes, grain textures
- Mobile: Blobs scale down, textures remain, fluid layout
- Rule: Nothing feels manufactured.

### TERMINAL / CYBERPUNK
- Background: #0a0a0a or deep navy
- Accents: #00ff88, #ff0055, #00d4ff
- Typography: JetBrains Mono, IBM Plex Mono
- Mobile: Reduce density, keep glow, monospace at 14px min
- Rule: Data is aesthetic. Density is beauty.

### RETRO / VINTAGE
- Faded colors, slab serifs, halftone textures
- Mobile: Poster composition stacks gracefully to single column
- Rule: Every element from a specific decade.

### FRAMEWORK
| Project Type              | Recommended Stack                    |
|---------------------------|--------------------------------------|
| Simple landing page       | HTML + CSS + Vanilla JS              |

### ANIMATION LIBRARIES
- GSAP + ScrollTrigger  — Scroll sequences, timelines, pinning
- Framer Motion         — React spring physics, layout animations
- Lenis                 — Smooth scroll inertia (use on almost all)
- Three.js / R3F        — 3D, WebGL, particles
- Anime.js              — Lightweight for simpler projects

### COMPONENT & STYLE LIBRARIES
- Tailwind CSS          — Utility-first base (responsive built-in)
- shadcn/ui             — Headless, highly customizable
- Radix UI              — Accessible headless primitives
- Mantine               — Full-featured, beautiful defaults
- Aceternity UI         — Premium animated components
- Magic UI              — Animated landing page components
- DaisyUI               — Tailwind themes and components

### ICON LIBRARIES
- Lucide React, Phosphor Icons, Tabler Icons

### DATA VISUALIZATION
- Recharts, D3.js, Tremor

### FONTS
- Always variable fonts. Google Fonts or Fontsource.
- font-display: swap always. Never base64.

## SECTION D — SCROLL & EXPERIENCE TYPES
═══════════════════════════════════════════════════════════════

### Standard Scrollable
- Lenis for smooth inertia
- Sticky nav with scroll-aware state
- Back-to-top on long pages

### Scrollytelling
- GSAP ScrollTrigger.pin() for desktop
- Mobile: convert to stacked vertical sections (no pinning)
- Progress bar shows story position

### Parallax
- Max 3 layers. CSS transform translateY only.
- DISABLE on mobile — causes jank and motion sickness

### Horizontal Scroll
- Show partial next element as affordance
- Keyboard navigable (arrow keys)
- Touch swipe supported

### Immersive / Full-Screen
- 100dvh sections (not 100vh — avoids mobile browser chrome issues)
- Smooth section transitions

## SECTION E — COMPLETE RESPONSIVE SYSTEM
═══════════════════════════════════════════════════════════════

MOBILE-FIRST IS THE LAW.
Write base styles for mobile. Layer complexity upward.
NEVER write desktop styles first and try to shrink down.

### BREAKPOINT SYSTEM

```css
/* The 7 breakpoints — cover every real device */
/* xs  */ @media (min-width: 320px)  { } /* Small phones: iPhone SE, budget Android */
/* sm  */ @media (min-width: 480px)  { } /* Large phones: iPhone Plus, Galaxy */
/* md  */ @media (min-width: 768px)  { } /* Tablets: iPad, Android tablet portrait */
/* lg  */ @media (min-width: 1024px) { } /* Laptops: iPad landscape, 13" laptops */
/* xl  */ @media (min-width: 1280px) { } /* Desktop: 15" laptops, standard monitors */
/* 2xl */ @media (min-width: 1536px) { } /* Large desktop: 1440p monitors */
/* 3xl */ @media (min-width: 1920px) { } /* Full HD / 4K monitors */
```

### SCREEN-BY-SCREEN LAYOUT RULES

#### 320px–479px — Small Mobile (iPhone SE, budget phones)
LAYOUT:
- Single column always. No exceptions.
- Full-width containers (100% - 32px padding, 16px each side)
- No horizontal scroll ever
- Stack everything vertically
- Cards: 100% width, stacked
- Images: 100% width, aspect-ratio locked

NAVIGATION:
- Hamburger menu or bottom navigation bar
- No horizontal nav links visible
- Logo left, hamburger right — standard expectation
- Mobile menu: full-screen overlay or bottom sheet
- Nav height: 56px

TYPOGRAPHY:
- Hero/Display: clamp(1.75rem, 8vw, 2.25rem)
- H1: clamp(1.5rem, 6vw, 2rem)
- H2: clamp(1.25rem, 5vw, 1.5rem)
- Body: 15px minimum — never below
- Line height: 1.6 for body

BUTTONS:
- Minimum height: 48px (touch target)
- Minimum width: 48px
- Full-width CTAs for primary actions
- Padding: 14px 24px minimum
- Stack buttons vertically if multiple

SPACING:
- Section padding: 48px vertical, 16px horizontal
- Component gap: 16px minimum
- Card padding: 20px

FEATURES TO DISABLE:
- Parallax effects
- Cursor custom effects
- Hover-only interactions (must have tap equivalent)
- Multi-column grids

---

#### 480px–767px — Large Mobile (iPhone Pro Max, Galaxy Ultra)
LAYOUT:
- Still single column for content
- Some 2-column allowed for small components (icon + text pairs)
- Max content width: 100% with 24px side padding

NAVIGATION:
- Still hamburger / bottom nav
- Nav height: 60px

TYPOGRAPHY:
- Hero: clamp(2rem, 8vw, 2.75rem)
- H1: clamp(1.75rem, 6vw, 2.25rem)
- Body: 16px

BUTTONS:
- Min height: 48px
- Primary CTA still full-width
- Secondary buttons can be inline

SPACING:
- Section padding: 56px vertical, 24px horizontal
- Card padding: 24px

---

#### 768px–1023px — Tablet (iPad, Android tablet portrait)
LAYOUT:
- 2-column grid for cards and features
- Sidebar appears at 240px (collapsible)
- Max content width: 720px centered
- Images can float alongside text

NAVIGATION:
- Can show top navigation with 4–5 links max
- OR keep hamburger if nav has many items
- Nav height: 64px
- Logo + nav links + CTA button in one row

TYPOGRAPHY:
- Hero: clamp(2.5rem, 5vw, 3.5rem)
- H1: clamp(2rem, 4vw, 2.75rem)
- Body: 16–17px

BUTTONS:
- Min height: 44px
- Auto width based on content
- Primary CTA: fixed width or inline

GRID:
- 2-column: grid-template-columns: repeat(2, 1fr)
- Gap: 24px

SPACING:
- Section padding: 64px vertical, 40px horizontal

FEATURES TO ENABLE:
- Hover states (hybrid — some tablets have mouse)
- Side-by-side comparisons
- Collapsible sidebar

---

#### 1024px–1279px — Laptop (13"–14" screens, iPad landscape)
LAYOUT:
- 2–3 column grids
- Sidebar fully visible at 260px
- Max content width: 960px
- Full desktop navigation

NAVIGATION:
- Full horizontal nav
- Nav height: 70px
- Dropdown menus if needed

TYPOGRAPHY:
- Hero: clamp(3rem, 4vw, 4.5rem)
- H1: clamp(2.25rem, 3vw, 3rem)
- Body: 17px

BUTTONS:
- Min height: 44px
- Comfortable padding: 14px 28px

GRID:
- 3-column: grid-template-columns: repeat(3, 1fr)
- Gap: 28px

SPACING:
- Section padding: 80px vertical, 48px horizontal

FEATURES TO ENABLE:
- Custom cursor effects
- Parallax (gentle, max 3 layers)
- Hover card reveals
- Multi-level dropdowns

---

#### 1280px–1535px — Desktop (15"–24" standard monitors)
LAYOUT:
- 3–4 column grids
- Max content width: 1200px centered
- Full layout complexity unlocked

NAVIGATION:
- Full nav with all links
- Mega menus if required
- Nav height: 72–80px

TYPOGRAPHY:
- Hero: clamp(4rem, 5vw, 6rem)
- H1: clamp(2.75rem, 3.5vw, 3.5rem)
- Body: 17–18px

GRID:
- 4-column: grid-template-columns: repeat(4, 1fr)
- Gap: 32px

SPACING:
- Section padding: 96px vertical, 64px horizontal

---

#### 1536px–1919px — Large Desktop (1440p monitors)
LAYOUT:
- Max content width: 1400px centered
- 4–5 column grids where appropriate
- More whitespace — content breathes

TYPOGRAPHY:
- Hero: clamp(5rem, 6vw, 8rem)
- Body: 18px

SPACING:
- Section padding: 120px vertical, 80px horizontal

---

#### 1920px+ — Full HD / 4K (Large monitors, TV displays)
LAYOUT:
- Max content width: 1600px centered — NEVER stretch to full screen
- Background fills edge to edge, content stays centered
- Larger decorative elements fill the extra space

TYPOGRAPHY:
- Hero: clamp(6rem, 7vw, 10rem)
- Body: 18–20px

SPACING:
- Section padding: 140px vertical, 100px horizontal

RULE: Content should never feel lost at 4K.
Use full-bleed backgrounds, decorative elements, or wider sidebars
to fill space rather than stretching content.

### TOUCH & POINTER RULES

```css
/* Touch targets — Apple HIG + Google Material minimum */
.interactive {
  min-height: 44px;
  min-width: 44px;
}

/* Remove hover effects on touch devices */
@media (hover: none) and (pointer: coarse) {
  .card:hover { transform: none; }
  .button:hover { background: var(--color-primary); }
  /* Keep only tap/active states */
  .button:active { transform: scale(0.97); }
}

/* Apply hover only on true pointer devices */
@media (hover: hover) and (pointer: fine) {
  .card:hover { transform: translateY(-4px); }
}
```
